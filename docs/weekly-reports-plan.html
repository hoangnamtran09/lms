Model & Migration

Bảng weekly_reports trong PostgreSQL + AutoMigrate

~15 phút

📄 backend/internal/reports/model.go

type WeeklyReport struct {
    ID          string    `gorm:"primaryKey;size:36" json:"id"`
    UserID      string    `gorm:"size:36;not null;index:idx_user_week" json:"userId"`
    WeekStart   string    `gorm:"size:10;not null;index:idx_user_week" json:"weekStart"`
    WeekEnd     string    `gorm:"size:10;not null" json:"weekEnd"`
    ReportJSON  string    `gorm:"type:jsonb" json:"reportJson"`
    AIMessage   string    `gorm:"type:text" json:"aiMessage"`
    CreatedAt   time.Time `json:"createdAt"`
}

🔧 backend/cmd/server/migrate.go

Thêm &reports.WeeklyReport{} vào danh sách AutoMigrate.

func migrate(db *gorm.DB) error {
    return db.AutoMigrate(
        // ... existing models ...
        &reports.WeeklyReport{},  <-- THÊM
    )
}

Unique composite index (user_id, week_start) tránh trùng report.

Phase 2

AI Prompt — Weekly Report

Thêm prompt mới vào backend/internal/ai/prompts.go

~20 phút

📝 Prompt structure

System prompt:

"Bạn là chuyên gia phân tích học tập. Phân tích dữ liệu tuần của học sinh và đưa ra nhận xét chi tiết bằng tiếng Việt, giọng tích cực, xưng 'mình' gọi 'bạn'."

User prompt — input data:

  • • Tổng giờ học (tuần này vs tuần trước)
  • • Daily breakdown (7 ngày)
  • • Bài tập đã hoàn thành + điểm TB
  • • Top 3 điểm yếu + trend
  • • Kim cương kiếm được
  • • Streak hiện tại

Output JSON yêu cầu:

{
  "title": "Tuần 20: Tiến bộ vượt bậc!",
  "overallAssessment": "positive" | "neutral" | "needsImprovement",
  "highlights": ["Hoàn thành 3/3 bài tập", "Streak 7 ngày"],
  "weaknessAnalysis": "Phân số vẫn là điểm yếu cần cải thiện...",
  "trendComparison": "Tăng 40% thời gian học so với tuần trước",
  "recommendations": [
    "Dành thêm 30 phút ôn Phân số",
    "Tiếp tục giữ streak hàng ngày"
  ],
  "coachMessage": "Chào bạn! Tuần này bạn đã học được 5 giờ... ✨"
}

Khác với Coach prompt (chỉ output text), prompt này yêu cầu JSON có cấu trúc để frontend hiển thị chart.

Phase 3

Service Layer

backend/internal/reports/service.go — Gather + Gen + Cache

~45 phút

📥 Data gathering — 5 nguồn

⏱ Study Sessions
study_sessions

SUM duration_seconds GROUP BY DATE, so sánh với tuần trước

📝 Submissions
submissions

COUNT + AVG(score) WHERE status = 'GRADED'

🔴 Weaknesses
weakness_profiles

Top 5 unresolved, error_count, improvement_score

💎 Diamonds
diamond_transactions

SUM(amount) WHERE created_at IN week

🔥 Streak
streaks

current_streak, last_study_date

🧩 Service methods

type Service struct {
    db    *gorm.DB
    aiSvc *ai.Service
}

func (s *Service) GetOrGenerate(ctx, userID, weekStart, weekEnd)  → cache-check + gen
func (s *Service) Generate(ctx, userID, weekStart, weekEnd)        → gatherData + callAI + save
func (s *Service) List(ctx, userID, limit)                         → list DESC by created_at
func (s *Service) GetByID(ctx, id)                              → single report
⚙️ Generate flow:
  1. Tính weekStart = Monday, weekEnd = Sunday
  2. Check DB: đã có report cho (userID, weekStart)? → return luôn (cache)
  3. Chưa có: gather 5 nguồn dữ liệu + dữ liệu tuần trước để so sánh
  4. Build prompt → gọi aiSvc.Chat() → parse JSON response
  5. Lưu WeeklyReport vào DB
  6. Return report

Handler & Router

HTTP endpoints + Chi router registration

~30 phút

📡 API Endpoints

GET /api/reports/list ?limit=10
GET /api/reports/{id} Chi tiết report
GET /api/reports/student/{studentId} Teacher/Parent xem của HS

🔌 Router wiring (router.go)

// Handlers struct
Reports *reports.Handler

// Init
reportsSvc := reports.NewService(db, aiSvc)
reportsH := reports.NewHandler(reportsSvc, db)

// Routes
r.Post("/api/reports/generate", h.Reports.Generate)
r.Get("/api/reports/list", h.Reports.List)
r.Get("/api/reports/{id}", h.Reports.Get)
r.With(permission(ResAnalytics, ActRead)).
    Get("/api/reports/student/{studentId}", h.Reports.GetForStudent)
🧠 Permission handling:
  • • STUDENT: chỉ xem report của chính mình (check claims.UserID)
  • • TEACHER: xem report của học sinh trong class (dùng classes + users.class_id)
  • • PARENT: xem report của con (dùng child_links)
